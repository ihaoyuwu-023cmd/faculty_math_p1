一、项目定位

本项目以教育部第四轮学科评估数学学科上榜高校为学校范围，建立一套可追溯、可复核、可更新的数学相关师资数据库、统计报告、轻量本地 App 和独立静态证据包。

项目回答以下问题：

1. 第四轮学科评估数学学科上榜学校及评级情况。
2. 每所学校实际承载数学学科的数学学院、数学系、数学中心及其他相关组织边界。
3. 各纳入单位的当前全职在任教师、职称和任职记录。
4. 教师公开披露的院士、四大青、四小青等国家级人才称号。
5. 教师研究方向及其 M1-M8 分类结构。
6. 校内多单位任职的单位级记录和学校级去重结果。
7. 来源冲突、访问受限、未找到个人主页、组织边界和名单数量异常。

本项目是基于公开官方资料的结构化统计，不评价教师学术水平，不对学校或教师作声誉判断，不推断官网没有公开的个人信息，不把统计结果解释为第四轮学科评估评级的因果原因。

二、最终时间口径

1. 第四轮学科评估评级是历史评级，只作为学校分组变量。
2. 师资统计使用统一采集基准日，当前数据版本以数据库中记录的快照日期为准；当前发布数据快照日期为 2026-07-18，原始采集计划基准日为 2026-07-13，后续更新必须在版本说明中明确日期。
3. 当前教师状态以采集时点的学校、学院或单位官方信息为依据。
4. 不把当前教师数量、人才称号数量直接解释为第四轮评估结果的成因。
5. 若要重建第四轮评估时期的历史师资，应单独建立历史版本项目，不能与当前快照混合。

三、学校范围与排除学校

1. 学校基准名单以教育部或学位中心发布的第四轮学科评估数学学科结果为唯一基准。
2. 不用第三方排名补充或删除上榜学校。
3. 学校更名、合并或机构调整时，保留历史名称、当前标准名称和名称映射关系。
4. 当前项目不纳入军校，明确包括中国人民解放军陆军工程大学等军校；报告说明中统一标注为“军校”。
5. 中国科学院大学不计入本次统计，报告说明中明确列出“中国科学院大学不计入本次统计”。
6. 被排除的学校不进入学校总数、教师总数、人才总数、研究方向总数或学校排序。
7. 排除记录仍可保留在内部学校边界表中，便于说明排除理由，但不得混入主统计。

四、组织边界规则

4.1 单位分层

每所学校先建立相关组织候选清单，再决定是否纳入，不能默认只有数学学院。

U1 核心数学单位：
数学学院、数学科学学院、数学系、数学研究院、数学中心，或官网明确承担数学学科建设主体职责的单位。

U2 相关纳入单位：
统计学院、系统科学学院、交叉学院、交叉研究院、校区文理学院或其他单位。只有在学校或单位官网明确证明其承担数学学科建设、数学一级学科培养、数学学位点招生、数学学科教师任职或数学学科共同建设时，才可纳入主统计。

U3 候选但不纳入单位：
仅因论文使用数学方法、名称中含统计或数据、承担一般数学课程，或存在个别数学相关研究人员，但没有数学学科组织归属证据的单位。U3 只保留核查记录，不进入主统计。

4.2 纳入证据

U2 单位至少需要一项明确的官方证据：

1. 学校或学院官网明确写明共同建设、支撑或参与数学学科。
2. 官方文件明确该单位承担数学一级学科或相关二级学科培养。
3. 学校官方文件明确该单位内的固定教师团队参与数学学科建设。
4. 官方招生、导师或学位点页面明确该单位属于数学学科培养体系。
5. 官方组织架构和学科建设页面能够闭合单位与数学学科之间的关系。

百度百科、搜索结果和第三方页面只用于组织结构外部对照，不能单独决定纳入。官网与百度对照不一致时，以当前学校官网为主，并在数据异常中保留来源冲突。

4.3 多单位关系

1. 同一学校的数学学院和数学中心等多个正式纳入单位，分别建立单位记录。
2. 同一人在多个正式纳入单位的教师名册中出现时，各单位分别统计任职记录。
3. 学校总体汇总使用唯一人员 ID 去重，并额外给出任职记录数、多单位人员数和多单位所属清单。
4. 兼职、双聘、联合聘任、跨院系合作、客座和访问关系不视为正式多单位任职，不进入对应单位主统计。
5. 教师报告不显示未经官网确认的教研室或下属组织；已核实的组织信息可以在内部数据库保留，当前教师报告以院系或单位为主。
6. 若学院没有教研室或下属一级机构，不得根据名称、惯例或页面格式虚构教研室。
7. 页面出现“分析”“应用数学”“山西大学数学系”等异常教研室名称时，必须回到学校组织架构、学院师资页和个人页核验；不能直接作为组织字段写入。
8. 核实后仍不能确认的下属组织，统一不显示，必要时在数据异常中说明。

五、人员纳入与排除规则

5.1 纳入主统计

只纳入采集时点仍为全职在任的教学科研人员和正式全职研究系列人员，包括教授、副教授、讲师、助理教授、研究员、副研究员等。兼任院长、系主任等行政职务，但仍明确具有正式全职教师或研究岗位的人员，可以纳入。

5.2 不纳入主统计

1. 兼职、客座、访问、荣誉职位。
2. 退休、荣休教师。
3. 博士后、学生。
4. 纯行政人员。
5. 实验技术人员、教辅人员。
6. 仅有项目参与、校外合作或非正式双聘关系的人员。
7. 已离职、已调离或无法确认当前在任状态的人员。
8. 只有旧个人页而当前名册明确不再任职的人员。

5.3 状态冲突

官网人员名册、个人主页、新闻或旧页面状态不一致时：

1. 优先使用当前学院名册和当前单位页面判断在任状态。
2. 个人主页主要用于研究方向、简介、项目、论文和人才信息。
3. 不能确认当前全职在任的人员不进入主统计，标记“待核实”或“非当前全职在任”。
4. 不因页面仍可访问就默认教师仍在职。
5. 不因个人页显示“on leave”就直接按正常在任统计；应保留状态说明并按当前官方状态决定是否排除。

六、数据模型与唯一 ID

采用 SQLite 作为规范化主数据库，静态前端通过只读 API 访问数据。数据库中必须至少保留以下逻辑实体：

1. schools：学校 ID、标准名称、历史名称、评级、官方评级来源、学校顺序、是否纳入。
2. units：单位 ID、学校 ID、单位名称、U1/U2/U3、纳入状态、纳入理由、官网来源。
3. persons：唯一人员 ID、姓名、英文名或别名、消歧信息。
4. appointments：人员 ID、学校 ID、单位 ID、职称、岗位系列、任职类型、在职状态、主统计标记、核验日期。
5. research_directions：人员或任职 ID、方向原文、展示方向、主 M 分类、次 M 分类、分类依据、置信度、复核状态。
6. talent_titles：人员 ID、称号原文、规范名称、T1-T4 类别、项目全称、年份、状态、证据 ID。
7. sources：来源 ID、原始 URL、标题、发布机构、来源等级、访问日期、页面状态、静态证据路径、SHA-256。
8. issues：异常 ID、学校、单位或人员、异常类型、说明、证据路径、处理结论、是否进入前端。
9. collection_runs：采集批次、版本、开始和结束时间、规则版本、统计快照日期。

每个关键事实必须可以追溯到一个或多个 source_id。规范化字段不得覆盖原始字段。应用中的学校级去重只改变汇总方式，不删除单位级任职记录。

七、研究方向分类规则

7.1 M 分类

M0 未公开或无法分类：来源未披露、只写“数学”、证据冲突或无法可靠归类。
M1 基础数学：逻辑、代数、数论、几何、拓扑、分析、函数论、动力系统等。
M2 计算数学：数值分析、数值代数、科学计算、计算几何、高性能计算等。
M3 概率论与数理统计：概率、随机过程、随机分析、数理统计及其分支。
M4 应用数学：微分方程、反问题、数学建模、数学物理、金融数学、生物数学等。
M5 运筹学与控制论：最优化、运筹、控制、博弈、决策、调度与组合优化等。
M6 系统科学：系统理论、复杂系统、网络科学、系统动力学等。
M7 数学交叉：机器学习理论、数据科学理论、信息与编码、密码数学、量子信息等。
M8 其他：保留原 M8 分类代码，但前端显示名称统一为“其他”。当前不改变历史分类代码，只替换显示名称。

7.2 证据优先级

1. 教师个人主页明确的“研究方向”“研究兴趣”“主要研究领域”等原文字段优先级最高。
2. 个人简介中的明确方向、科研项目、个人论文和官方招生导师页面用于补充和复核。
3. 学院名册、教研室名称、职称和课程名称不能单独作为个人方向。
4. 个人页子栏目必须完整检查，不只读取首页首屏；包括个人简介、研究兴趣、科研项目、论文、招生导师、获奖和相关栏目。
5. 学院研究生教育、研究生招生和导师页面可以作为正式校内官方补充来源，尤其适用于个人主页缺失或方向字段不完整的教师。
6. 外部个人网站、GitHub Pages、Google Sites、搜索摘要、百科、Google Scholar、OpenAlex、Crossref、出版社页面不作为本轮方向结论的主证据；校内官网确实没有信息时保留 M0。

7.3 原文与总结

1. 如果官网存在明确的研究方向或研究兴趣原文字段，教师报告使用该字段的准确内容，不为了省字数随意删改，不把明确方向误写成“推断”。
2. 个人报告中的研究方向可以将明确原文做轻量格式整理，例如去掉编号、合并同义标点，但不得改变语义。
3. 如果官网没有明确方向字段，才根据完整个人页面和本人绑定科研信息进行简短总结，并在备注写“研究方向由主页信息总结（可信度：高/中/低）”。
4. 无明显方向或未区分时，研究方向写“未公开/未细分”，备注写“未公开/未细分”。
5. 研究方向不设机械字数上限；只有在前端展示时进行换行、折叠或合理排版，不能为压缩页面而丢失信息。
6. 研究方向展示应为短语或清晰句子，不粘贴整段个人简介、论文摘要或项目列表。
7. 明确字段“生物数学（一）传染病建模（二）复杂网络传播动力学”应记为明确方向，例如“生物数学：传染病建模、复杂网络传播动力学”，备注为“无”，而非“研究方向由主页信息总结”。
8. 明确字段“主要研究方向：调和分析，包括函数空间实变理论、算子有界性、插值理论等”应归纳为“调和分析：函数空间实变理论、算子有界性、插值理论”，而不是把后续解释性长段落作为方向。
9. “应用数学”可计入 M4，但备注“官网仅给出宽口径方向，未细分具体研究主题”；“数学”仍计入 M0。
10. 所有 M0 必须逐人复核，重点检查个人页子页面、研究生招生页和导师介绍页；复核后仍无可靠方向才保留 M0。

7.4 推断方向的条件

个人页没有明确方向字段时，只有在以下条件同时满足时才允许总结：

1. 至少三项本人绑定、主题一致的校内官网科研成果或项目。
2. 至少一项有明确年份，优先使用 2021-2026 年资料。
3. 页面明确显示这些成果属于本人，而非课题组共用列表。
4. 原文题目、年份、官方 URL 和静态证据均已保存。
5. 分类理由和可信度写入数据库。

不满足条件时保留 M0，不编造、不按教研室名称推断。

八、人才帽子规则

8.1 主统计范围

主统计只保留 T1-T4：

T1：国内两院院士，包括中国科学院院士、中国工程院院士。
T2：国家级重大人才计划，按官方公开完整项目名和具体类别记录。
T3：国家级科学基金人才项目，重点包括国家杰出青年科学基金、国家优秀青年科学基金等，按官方完整名称区分。
T4：教育部及其他中央部门人才项目，只有明确属于本口径的项目才纳入。

8.2 明确排除

1. T5、T6 不再作为当前主统计类别或前端字段。
2. 国际科学院院士、国际学会 Fellow 等国际学术荣誉不进入 T1-T4 计数。
3. 省级、直辖市级人才项目不进入 T1-T4 计数。
4. 校级人才岗位、校级荣誉不进入 T1-T4 计数。
5. 教学奖励、洪堡基金、国家优秀自费留学生奖学金等，不因其重要性自动归入 T1-T4。
6. 省级、校级和国际称号如未被当前数据结构保留，不要求在主 App 中显示。

8.3 记录要求

1. 记录人才项目全称、具体类别、年份、证据原文和来源 URL。
2. 只有官方公开证据或学校官方教师介绍明确披露时才计入。
3. 只写“国家高层次人才”“国家级青年人才”等泛称而没有具体类别时，不能强行映射，进入异常或待核实记录。
4. 一人多个称号分别记录；学校汇总的持有人数按唯一人员去重。
5. 同一称号在多个官方页面重复出现时合并条目，但保留全部来源。
6. 未公开年份写“未公开”，不根据履历猜测。
7. 统计总数必须能从人才明细重新计算，避免出现总览与教师报告不一致。

九、来源、静态证据与访问失败

9.1 来源等级

S1：教育部、学位中心、基金委、国家部委和人才项目官方公告。
S2：学校、学院、校内研究机构官方目录和个人主页。
S3：学校或学院官方项目、论文、成果、招生和导师页面。
S4：权威媒体、学会新闻、会议简介，仅作辅助线索。
S5：百科、聚合站、搜索摘要和外部学术数据库，不作为方向结论证据。

9.2 固定证据

1. 每个个人主页、名册页、组织结构页、人才公告和冲突页面尽量保存为本地静态 HTML。
2. 静态文件保存原始页面内容或执行脚本后的完整 DOM；不得只保存报告中的一段摘录。
3. 保留原始 URL 与静态 HTML 的对应表 `url-map.csv`。
4. 每个文件记录访问日期、页面标题、来源类型、页面状态和 SHA-256。
5. App 默认打开官网原始 URL；只有在网页端输入静态证据包绝对路径后，官方来源链接才切换为本地证据。
6. 输入的是证据包文件夹绝对路径，不是 URL、Markdown 链接或单个 HTML 文件。
7. 没有配置证据包时，App 仍可正常使用，来源链接打开官网。
8. 验证码、登录限制、必须人工访问的数据库、外部个人页面连接失败等情况不反复冲击；记录“访问受限”或“未找到个人页面”。
9. 个人页访问失败不能用外部站点替代；如果学院官网已有完整官方信息，可以使用学院官网；否则保留 M0 或未找到个人页面。

十、异常与冲突规则

数据异常不是静默修正，而是可追踪的数据对象。前端“数据异常”页面只显示问题说明、学校和证据，不显示不需要的“涉及人员”“严重程度”“状态”等字段。

需要记录的异常类型包括：

1. 官方汇总数与实名名单不一致。
2. 名单存在但个人页未找到。
3. 个人页访问受限、403、连接失败或页面失效。
4. 官网个人页、学院名册、组织页面的任职状态冲突。
5. 学校官网和外部组织对照信息不一致。
6. 组织或教研室名称异常、无法确认真实存在。
7. 人才称号类别无法确定。
8. 研究方向来源不足、方向过宽或分类存在争议。
9. 静态证据未保存、映射缺失或文件校验失败。

10.1 名单与汇总数冲突

重庆师范大学、沈阳师范大学、湖南师范大学、浙江工业大学、南昌大学、四川师范大学等出现过官方汇总数与实名名单不一致的情况。处理原则统一为：保留实名名单、保留官方汇总数和双方来源，不推算、不虚构缺失姓名，在数据异常中列出差额和证据路径。

10.2 访问受限

浙江工业大学 403 页面、上海交通大学部分个人页连接失败等情形，记录具体 URL、失败状态和访问日期，不反复请求，不把失败误写成“没有个人页面”。

10.3 组织边界排除

华东师范大学统计学院、厦门大学经济学院统计学与数据科学系等，若没有数学主统计边界的官方证据，不纳入数学主统计。河南师范大学惠昌常等若不是当前全职在任，即使曾在数学学院页面出现，也不纳入主统计；必须记录被排除的原因和证据。

十一、采集与复核工作流

阶段 0：规则冻结。
确认学校范围、排除学校、人员状态、组织边界、M 分类、人才白名单、来源等级和输出字段。

阶段 1：学校基准表。
核对第四轮数学学科名单、评级、学校标准名、当前官网域名和学校排序。

阶段 2：组织边界检索。
通过学校官网、百度搜索和校内导航查找数学学院、数学中心、统计、系统科学、交叉和校区相关单位；逐一写明纳入、排除或待核实理由。

阶段 3：师资名册采集。
优先抓取完整师资名册，再逐人核验姓名、职称、在任状态、单位和个人页入口。分页名册必须逐页检查，不只读取首屏。

阶段 4：个人主页深度采集。
打开每个可用个人页的所有相关栏目，检查研究兴趣、个人简介、科研项目、论文、招生导师、获奖和联系方式。学院个人页缺失时，检查学院研究生教育、招生和导师页。

阶段 5：研究方向复核。
明确字段直接使用并轻量整理；无明确字段时按严格条件总结；所有 M0 逐人审阅；研究方向不因字数限制删改。

阶段 6：人才复核。
逐人检查院士、四大青、四小青，记录全称和来源；检查学校新闻中的遗漏，但必须回到官方教师页或项目公告确认。

阶段 7：证据固化。
下载、保存、校验 HTML，建立 URL 映射、来源索引和失败记录。

阶段 8：一致性验收。
重新计算单位人数、学校去重人数、M 分类和 T1-T4，检查重复、空值、链接、证据和页面字段。

阶段 9：发布。
生成只读 SQLite、轻量 App、报告、Excel 和独立证据包。数据更新必须从数据同步脚本和校验流程进入，不能直接手改 HTML。

十二、质量控制与验收标准

12.1 名单完整性

1. 学校基准名单覆盖率为 100%。
2. 每所学校有完成、资料不完整、来源冲突或受限等明确结果。
3. 每条主统计教师记录至少有一条当前官方任职来源。
4. 每个纳入 U2 单位有官方纳入证据。
5. 个人页能找到时，必须保存个人页 URL；静态包生成时应有对应 HTML。
6. 个人页找不到时写“未找到个人页面”；访问受限时写“官网个人页访问受限”，不能混用。

12.2 数量闭合

1. 单位明细数量必须可以重新计算。
2. 学校同时显示单位任职记录数和唯一人员数。
3. 校内多单位人员不能在去重统计中重复计数。
4. 组织边界不确定时，不为了闭合官方汇总数而补写姓名。
5. 官方汇总数只作为来源事实，不替代实名名单。

12.3 方向与人才

1. 所有方向记录有原文或明确的总结依据。
2. 所有 M0 逐人复核过，并有结论或复核说明。
3. M8 前端名称统一为“其他”。
4. 所有 T1-T4 记录有项目全称和来源。
5. 旧 T5/T6 不得出现在当前主 App 的人才统计、列名或总览计数中。
6. 总览人才计数与人才明细及教师报告一致。

12.4 证据验收

1. URL 映射表中的静态路径全部存在，或明确记录失败状态。
2. 抽查个人页、名册页和人才来源均可打开本地 HTML。
3. 静态文件与原 URL、教师或学校 ID 的关系可反查。
4. 页面内容变动后仍可用本地快照审阅，不依赖网页继续在线。
5. 证据包与轻量 App 可以分开分发。

十三、应用架构

13.1 轻量 App

轻量 App 由三个层次组成：

1. 前端层：静态 HTML、CSS、JavaScript，提供总览、学校、教师、数据异常、说明和设置页面。
2. 数据层：SQLite 只读数据库，保存完整统计结果和来源字段。
3. 服务层：Python 标准库本地 HTTP API；正式发布同时提供 exe，普通使用者不需要安装 Python。

当前轻量 App 目录：

13.2 静态证据包

证据包独立于轻量 App：

App 不默认依赖证据包。用户在网页设置中输入证据包绝对路径后，服务端将来源链接映射到本地静态 HTML；清除路径后恢复官网链接。

13.3 启动器

1. 双击根目录或 light-app 目录的启动 BAT 均应通过 Windows 默认浏览器打开 App。
2. 不要求用户安装 Chrome；系统有默认浏览器即可。
3. 启动器先检查既有同版本服务和端口，再决定复用或启动。
4. 每个启动实例必须固定一个端口，不能让旧进程占用新版本端口。
5. 端口被占用时自动寻找允许范围内的空闲端口，并把实际 URL 写入日志。
6. 启动后必须检查健康接口和数据库状态，确认服务真正监听后再打开浏览器。
7. 启动失败必须写入日志并返回明确错误，不能只打开空白页面或假定启动成功。
8. 当前轻量 App 默认使用 8766，项目根目录兼容启动器使用 8767；实际端口以启动日志和健康接口为准。
9. 启动器必须使用绝对路径、独立日志和唯一版本标识，解决重复进程、端口混用和旧版本缓存问题。

十四、前端功能与显示规则

1. 总览页面显示学校数量、任职记录数、去重教师数、T1-T4 人才记录和方向公开率。
2. 总览中保留采集日期标签。
3. 总览下方保留简短说明，明确军校不计入、中国科学院大学不计入；不再使用“ 不计入的军校 ”、“ 不计入的其他学校 ”等标题字样。
4. 数据异常作为左侧独立入口，显示学校顺序与学校页面一致；同一学校的多条问题按当前顺序逐条显示，不强行合并。
5. 异常页面的问题类型全部使用中文，去掉“涉及人员”“严重程度”“状态”列，只保留学校、问题类型、问题说明、证据和必要的处理说明。
6. 学校页提供教师报告入口；删除单位报告和不需要的饼图模块。
7. 教师报告字段为姓名、学校、院系/单位、职称、M 分类、研究方向、备注、T1-T4 人才帽子、官方来源。
8. M 分类采用紧凑两行显示：第一行代码，第二行分类名称；M8 名称显示“其他”。
9. 不使用彩色 M 标签，表格列宽稳定，文字居中或按字段类型统一对齐。
10. 学校、院系、职称列保持足够宽度；研究方向列使用剩余空间；备注、人才和来源列不挤压主要信息。
11. 长表格采用分页和正常页面滚动，不在单个内容框内制造不必要的横向滚动。
12. 底部提供翻页控件，当前页内容恰好容纳；窗口变窄时正常滚动，不遮挡列名和内容。
13. 多任职人员在学校或单位显示中按上下换行，学校总体标记多单位所属。
14. 来源按钮在未配置证据包时打开官网；配置证据包后打开本地静态证据。
15. 应用设置中的路径输入必须处理字符串和后端对象返回值，禁止显示“[object Object]”。

十五、数据更新流程

1. 数据库为发布 App 的只读运行数据，不直接在浏览器或 HTML 中编辑。
2. 更新前先备份当前数据库、记录版本号和变更摘要。
3. 由采集脚本更新原始数据、来源和问题记录。
4. 执行方向、人才、组织边界和状态复核。
5. 执行数据库完整性、引用完整性、数量闭合和证据映射检查。
6. 生成新的只读数据库和 App 构建版本。
7. 冷启动测试启动器、健康接口、默认浏览器、学校页、教师报告、异常页和证据路径配置。
8. 验收通过后才替换 light-app 的发布文件；失败则保留旧版，不覆盖可用版本。
9. 证据包更新不应强制重建轻量 App；两者通过 URL 映射和绝对路径配置连接。

十七、分发规则与隐私

1. 只分发轻量 App 时，分发完整的 light-app 文件夹；接收者使用 Windows 默认浏览器即可，不要求 Chrome，不要求 Python。
2. 需要本地证据时，另行分发 evidence-package 文件夹，并在 App 网页设置中输入其绝对路径。
3. 不要只分发 light-app\\app，因为数据库在 light-app\\data，启动器和配置也在上级目录。
4. 数据库随 App 分发，拥有 light-app 文件夹的人可以复制或读取数据库中的统计数据；它不具备数据库加密和访问控制功能。
5. 静态证据包可能很大，应根据接收者需求单独分发，不必与轻量 App 放在同一目录。
6. 删除旧目录前必须确认不再需要历史报告、原始数据库、处理脚本和回滚备份。

十九、方案变更原则

1. 任何统计口径变更必须修改数据字典、数据库版本和本方案，不直接改报告文字。
2. 任何分类规则变更必须保留旧分类版本，重新生成分类结果并说明差异。
3. 任何来源修正必须保留旧来源状态和修正理由。
4. 不为了减少 M0、填平学校汇总数或提高公开率而补写没有证据的事实。
5. 前端展示调整不得改变主数据库事实；数据修改和界面修改分开验收。
6. 只读发布包通过验收后才可作为正式版本，实验页面、旧 HTML、旧脚本和临时文件不得混入发布目录。

二十、结论

本项目最终采用“规范化 SQLite 数据库 + 只读轻量本地 App + 独立静态证据包”的结构。数据库负责事实和统计，轻量 App 负责查询、筛选、汇总和异常审阅，静态证据包负责长期保存和离线复核。三者通过人员 ID、单位 ID、来源 ID 和 URL 映射关联，但可以独立分发。

项目的核心原则是：官方证据优先、当前全职在任优先、组织边界逐校核验、个人页面深度检查、明确方向不误判、缺失信息不编造、来源冲突不强行闭合、校内重复在学校层去重、所有结论可追溯到固定证据。
